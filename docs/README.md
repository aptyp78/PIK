# OCR AI Server для сложных PDF документов

Продвинутый OCR сервер для распознавания сложных PDF документов с диаграммами, таблицами, схемами и многоязычным текстом.

## 🚀 Ключевые возможности

### 📊 **Обработка сложного контента**
- **Автоматическое определение областей**: текст, таблицы, диаграммы
- **Специализированные алгоритмы OCR** для разных типов контента
- **Извлечение структурированных таблиц** с форматированием в Markdown
- **Поддержка русского и английского языков**

### 🔧 **Продвинутая предобработка**
- **5 алгоритмов улучшения изображений**: контраст, бинаризация, морфология, деноизинг
- **Адаптивная обработка** в зависимости от типа контента
- **Мультиугловое распознавание** для наклонного текста
- **Высокое разрешение** (до 5x) для захвата мелких деталей

### 🛡️ **Безопасность и надежность**
- **Валидация PDF файлов** по magic number
- **Ограничение размера файлов** (100MB)
- **Детальное логирование** и обработка ошибок
- **Метрики производительности**

## 📦 Установка

### 1. Установка Python зависимостей
```bash
pip install -r requirements.txt
```

### 2. Установка Tesseract OCR

**macOS:**
```bash
brew install tesseract tesseract-lang
```

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install tesseract-ocr tesseract-ocr-rus tesseract-ocr-eng
```

**Windows:**
Скачайте и установите с https://github.com/UB-Mannheim/tesseract/wiki

### 3. Проверка установки
```bash
tesseract --version
```

## 🏃‍♂️ Запуск сервера

```bash
uvicorn ocr_server:app --host 0.0.0.0 --port 8000 --reload
```

Сервер будет доступен по адресу: `http://localhost:8000`

## 📖 API Endpoints

### Основные эндпоинты

#### `POST /convert` - Конвертация PDF в Markdown
```bash
curl -X POST "http://localhost:8000/convert?mode=complex&lang=eng+rus" \
     -F "file=@document.pdf"
```

**Параметры:**
- `mode`: `simple` | `standard` | `complex` | `max` (по умолчанию: `complex`)
- `scale`: разрешение рендеринга `2.0`-`5.0` (по умолчанию: `4.0`)
- `lang`: языки OCR, например `eng+rus` (по умолчанию: `eng+rus`)
- `angles`: углы поворота, например `0,90,270,180,-15,15`
- `corner`: обработка углов `1`|`0` (по умолчанию: `1`)

#### `POST /api/v1/marker` - Marker-совместимый API
Для интеграции с существующими клиентами Marker.

#### `GET /metrics` - Метрики сервера
Статистика производительности и использования.

### Режимы обработки

| Режим | Разрешение | Углы поворота | Описание |
|-------|------------|---------------|----------|
| `simple` | 2.0x | 0° | Быстрая обработка простых документов |
| `standard` | 3.0x | 0°, ±90°, 180° | Стандартная обработка |
| `complex` | 4.0x | 0°, ±90°, 180°, ±15° | Сложные документы с диаграммами |
| `max` | 5.0x | 0°, ±90°, 180°, ±15°, ±30°, ±45° | Максимальное качество |

## ⚡ Производительность

- **Среднее время обработки**: 5-15 сек на страницу (зависит от сложности)
- **Поддерживаемый размер**: до 100MB PDF файлов
- **Память**: ~500MB-2GB в зависимости от документа
- **CPU**: использует все доступные ядра для OCR

## 🐛 Устранение неполадок

### Частые проблемы

1. **Ошибка "tesseract not found"**
   ```bash
   # Проверьте путь к tesseract
   which tesseract
   # Добавьте в PATH если нужно
   export PATH="/usr/local/bin:$PATH"
   ```

2. **Медленная обработка**
   - Уменьшите режим с `max` на `complex` или `standard`
   - Проверьте доступную память

3. **Плохое качество распознавания**
   - Увеличьте разрешение (`scale=5.0`)
   - Добавьте больше углов поворота
   - Проверьте качество исходного PDF
